Modelo MVC

Sus siglas son de Modelo, Vista y Controlador, y es un patrón de arquitectura que divide una aplicación en 3 capas principales, su objetivo es organizar el código, separar responsabilidades y hacer que la aplicación sea más escalable, mantenible y fácil de probar, este se usa en muchos frameworks modernos como lo pueden ser, Laravel, ASP.NET MVC, Ruby on Rails, Django, Spring MVC, Angular, React, entre otros.

Capas del modelo mvc

  • Modelo: Representa los datos y la lógica de negocio, Ejemplo: Clase Usuario que guarda, busca o valida usuarios.
  • Vista: Es la capa que muestra la información al usuario, Ejemplo: Página usuario.blade.php que muestra la lista de usuarios.
  • Controlador: Actúa como intermediario entre Modelo y Vista, Ejemplo: UsuarioController que responde a la ruta /usuarios.

Este nos a ayuda mucho a:

  • Mantener el código organizado
  • Facilita el trabajo en equipo
  • Permite reutilizar componentes
  • Facilita las pruebas y el mantenimiento
  • Escala mejor para proyectos grandes

Ejemplos de uso:

Modelo:
<?php class Producto extends Model { protected $table = "productos"; } ?>
Vista:
<?php class ProductoController extends Controller { public function index() { $productos = Producto::all(); return view("productos.index", compact("productos")); } } ?>
Controlador:
<h1>Lista de productos</h1> <ul> @foreach($productos as $p) <li>{{ $p->nombre }} - {{ $p->precio }}</li> @endforeach </ul>
